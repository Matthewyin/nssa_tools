MLG Bug修复验证

验证MLG游戏中的两个关键bug修复效果。

🎨 Bug 1: 立方体颜色层级判断修复

问题描述：
有些立方体未置于下面的层级，但也被错误地渲染成了暗色。

问题原因：
颜色判断基于网格位置(col,row)，但覆盖判断基于像素重叠，两者标准不一致。

修复方案：
1. 统一使用基于像素重叠的判断标准
2. 创建 getOverlappingTiles() 函数计算实际重叠
3. 使用 isTopInOverlapArea() 判断是否为重叠区域的顶层
4. 相对深度基于实际重叠的更高层级立方体数量

预期效果：
✅ 所有最上面层级的立方体都是亮色
✅ 被覆盖的立方体按层级深度显示不同暗色
✅ 颜色判断与点击判断逻辑一致

📱 Bug 2: 手机端显示不全修复

问题描述：
手机端右侧的立方体被截断，无法完整显示。

问题原因：
1. 边界约束逻辑只考虑基础网格，忽略了层级偏移
2. 手机端边距设置不够优化

修复方案：
1. 改进边界约束逻辑，正确处理总宽度超出的情况
2. 优化手机端CSS边距：从8px减少到6px
3. 确保画布能使用全部可用宽度

预期效果：
✅ 手机端所有立方体都能完整显示
✅ 内容在画布中正确居中
✅ 不会出现右侧截断的问题

🔧 扩容道具功能权限说明

问题： 扩容道具功能是否需要登录？

回答： 扩容道具功能不需要登录

理由：
1. 扩容道具是游戏内的道具机制，属于游戏玩法的一部分
2. 不涉及敏感数据或跨用户功能
3. 游客模式下也应该能完整体验游戏功能
4. 道具使用记录可以保存在本地localStorage中

实现状态： ✅ 已支持游客模式使用扩容道具
